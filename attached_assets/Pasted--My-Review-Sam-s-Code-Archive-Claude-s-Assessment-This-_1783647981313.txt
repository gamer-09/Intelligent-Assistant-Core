## Group A — What you actually wrote yourself

Files: `login1.0.html`, `assign 2.html`, `hel.html`, `decrypt.html`,
`home/sun.html`, `basqq.html`, `dah.html`, `fuew.html`, `naw.html`,

**HTML/CSS — beginner, and it shows in a specific way.** You're positioning
everything with `margin-left: 490px`, `top: 90px`, `left: 450px` — absolute
pixel values eyeballed to look right at one specific screen size. This
breaks the second someone resizes the window or opens it on a different
screen. It's not "bad code" for what it was (quick personal test pages),
but it tells me you haven't yet internalized flexbox/grid as your default
reach for layout — which we already covered in the Tailwind lessons, but
knowing the utility class names isn't the same as reaching for `flex` and
`justify-between` automatically instead of a ruler-and-guess approach.

**JavaScript — functional, not clean.** Variable names like `sa`, `gas`,
`das`, `ya`, `ka`, `q`, `f` across files with no consistency — you clearly
understand loops, conditionals, and array methods (`.sort()`, `.length`,
string concatenation), but there's no separation of concerns; everything
lives in one inline `<script>` tag dumping straight to `document.write` or
`innerHTML`. That's fine for a one-off answer to a homework question. It's
not fine as a habit going into a degree where you'll be graded partly on
code organization.

**Python — this is your strongest group, honestly.** The `eval(input())`
pattern repeated across `icp assignment.py`, `learn icp.py`, `icp class
3.py` is clearly how your course is teaching input handling, and you're
using it correctly and consistently. Your logic (temperature conversion,
random number matching, squares/exponents) is correct. The gap here isn't
understanding — it's that everything is script-level, top-to-bottom, no
functions, no reuse. That's expected at this stage, not a red flag.

**Java — barely past "hello world," and that's an accurate, not unkind,
assessment.** `checkmate.java`, `class2.java`, `class_personal.java` are
short, simple class/method demonstrations. Nothing wrong with them for
where you are, but there's no evidence yet of you handling multiple
interacting classes, inheritance, or anything beyond a single method call.
Given UNB's SWE program will hit Java or a similar OOP language hard early,
this is the weakest link of the four languages here.

**My honest skill read, Group A only:** JavaScript and HTML/CSS —
beginner, comfortable with syntax but not structure. Python — beginner,
but your strongest, most consistent group. Java — earliest beginner stage,
thinnest evidence.

## The one thing I'd tell you to fix first

Function decomposition. It's the single most repeated gap across every
one of your own files, in every language — everything is one long
top-to-bottom script. That's the fastest, highest-leverage thing to fix
before September, because it's not language-specific: once you get in the
habit of "one function, one job" in Python, it transfers directly to Java,
JavaScript, and whatever else UNB throws at you.
